## 故障排除

如果你在 SSH 连接过程中遇到问题，可以尝试以下故障排除步骤。

### 检查 SSH 服务状态

如果你使用远程连接（通过局域网而非直连显示器），请先确保 SSH 服务正在运行：

<NewCodeBlock tip={`radxa@${props?.board ?? 'device'}$`} type="device">

```bash
sudo systemctl status ssh
```

</NewCodeBlock>

如果服务未运行，使用以下命令启动：

<NewCodeBlock tip={`radxa@${props?.board ?? 'device'}$`} type="device">

```bash
sudo systemctl start ssh
sudo systemctl enable ssh
```

</NewCodeBlock>

:::tip
如果 `ssh` 服务名无效，可尝试 `sshd`：`sudo systemctl status sshd`
:::

### 检查 SSH 配置

确保 SSH 配置允许密码认证：

<NewCodeBlock tip={`radxa@${props?.board ?? 'device'}$`} type="device">

```bash
sudo grep -v "^#" /etc/ssh/sshd_config | grep PasswordAuthentication
```

</NewCodeBlock>

预期输出应为 `PasswordAuthentication yes`。如果不是，编辑配置文件：

<NewCodeBlock tip={`radxa@${props?.board ?? 'device'}$`} type="device">

```bash
sudo nano /etc/ssh/sshd_config
```

</NewCodeBlock>

找到并确保以下行未被注释且值为 `yes`：

```
PasswordAuthentication yes
```

保存后重启 SSH 服务：

<NewCodeBlock tip={`radxa@${props?.board ?? 'device'}$`} type="device">

```bash
sudo systemctl restart ssh
```

</NewCodeBlock>

### 检查网络和防火墙

确保防火墙未阻止 SSH 连接（默认端口 22）：

<NewCodeBlock tip={`radxa@${props?.board ?? 'device'}$`} type="device">

```bash
# 检查 iptables 规则
sudo iptables -L -n | grep -E "22|ssh"

# 或检查 nftables
sudo nft list ruleset 2>/dev/null | grep -E "22|ssh"
```

</NewCodeBlock>

### 验证 IP 地址

确保你使用了正确的 IP 地址：

<NewCodeBlock tip={`radxa@${props?.board ?? 'device'}$`} type="device">

```bash
ip addr show | grep inet
```

</NewCodeBlock>

在 PC 端也可以用以下方式确认设备 IP：

<NewCodeBlock tip="PC$" type="host">

```bash
# Windows
arp -a | findstr "192.168"

# Linux/macOS
arp -a | grep 192.168
```

</NewCodeBlock>

### 检查 SSH 日志

查看 SSH 服务日志获取详细错误信息：

<NewCodeBlock tip={`radxa@${props?.board ?? 'device'}$`} type="device">

```bash
sudo journalctl -u ssh -b --no-pager | tail -30
```

</NewCodeBlock>

### 常见问题

#### 密码错误

确保使用正确的用户名和密码。Radxa OS 默认用户名为 `radxa`，密码为 `radxa`。

如果忘记密码，可以通过串口连接或连接显示器后重新设置：

<NewCodeBlock tip={`radxa@${props?.board ?? 'device'}$`} type="device">

```bash
sudo passwd radxa
```

</NewCodeBlock>

#### SSH 公钥权限问题

如果之前配置过 SSH 公钥登录，可能因权限问题导致失败。检查服务端权限：

<NewCodeBlock tip={`radxa@${props?.board ?? 'device'}$`} type="device">

```bash
# 检查 .ssh 目录权限
ls -la ~/.ssh

# 正确权限应为 700
chmod 700 ~/.ssh

# 检查 authorized_keys 权限（应为 600）
chmod 600 ~/.ssh/authorized_keys
```

</NewCodeBlock>

#### Windows SSH 客户端问题

如果使用 Windows SSH 客户端遇到问题：

1. 尝试使用 Windows 自带 SSH 客户端（PowerShell 或 CMD）
2. 尝试使用 [Tabby](https://tabby.sh/)、[PuTTY](https://www.putty.org/) 或 [MobaXterm](https://mobaxterm.mobatek.net/)
3. 检查 Windows 防火墙设置，确保未阻止 SSH 连接

#### 连接超时

1. 确认设备与 PC 在同一局域网
2. 确认 IP 地址正确（设备 IP 不能是 `127.0.0.1`）
3. 尝试 ping 设备 IP 确认网络连通

<NewCodeBlock tip="PC$" type="host">

```bash
ping 192.168.x.x
```

</NewCodeBlock>

### 重置 SSH 配置

:::warning
以下操作会中断 SSH 连接。如果你是远程连接，请确保有其他接入方式（如串口）后再执行。
:::

如果以上步骤都无法解决问题，可以重置 SSH 配置：

<NewCodeBlock tip={`radxa@${props?.board ?? 'device'}$`} type="device">

```bash
# 备份当前配置
sudo cp /etc/ssh/sshd_config /etc/ssh/sshd_config.backup

# 恢复默认配置
sudo dpkg-reconfigure openssh-server
```

</NewCodeBlock>

重启后重新配置 SSH 连接。

### 获取帮助

如果问题仍然存在，请在 GitHub 上提交 Issue 并提供以下信息：

- SSH 服务状态输出 (`sudo systemctl status ssh`)
- SSH 配置中 `PasswordAuthentication` 的设置
- 防火墙状态
- 从客户端尝试连接时的完整错误信息
- SSH 日志（`sudo journalctl -u ssh -b --no-pager | tail -30`）
