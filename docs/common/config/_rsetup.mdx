# Rsetup 使用指南

## 简介

`rsetup` 是一个适用于 Radxa OS 的系统配置工具，提供了友好的 TUI（文本用户界面）来管理系统设置、硬件配置和网络连接等功能。

## 启动 rsetup

要启动 `rsetup`，请在终端中执行以下命令：

```bash
rsetup
```

或从系统菜单中找到 `rsetup` 应用启动。

:::tip
`rsetup` 运行时需要管理员权限，请确保你拥有 `sudo` 权限。

如果以普通用户身份启动，系统会自动提示使用 `pkexec` 或 `sudo` 提权。

下面列出的选项仅供参考，请以实际程序输出为准。
:::


## 界面操作

### 基本操作

你可以使用下列按键进行操作：

- **上下键** (`Up`/`Down`): 在菜单项之间切换，当前选中的项目会高亮显示
- **Tab 键**: 在菜单区域和命令区域之间切换
- **左右键** (`Left`/`Right`): 在 `<Ok>` 和 `<Cancel>` 命令之间切换
- **Enter 键**: 执行当前选中的命令或菜单项
- **Space 键**: 选择选项（用于多选框 `[ ]` 或单选框 `( )`）
- **Esc 键**: 返回到上一级菜单

使用 `Up` 和 `Down` 键来切换选项。当前选中的项目会高亮显示。

使用 `Tab` 键来切换菜单区域和命令区域。

使用 `Left` 和 `Right` 键来切换 `<Ok>` 和 `<Cancel>` 命令。使用 `Enter` 键来执行选中的命令。
当命令没有被选中时按下 `Enter` 键，会立即执行 `<Ok>` 命令。

使用 `Space` 键选择选项。选项会包含 `[ ]` （多项选择）或 `( )` （单项选择）前缀来显示当前的选择状态。

:::tip
使用 `Enter` 键不会选择选项。它会直接执行 `<Ok>` 命令。
:::

使用 `Esc` 键回到前一个菜单。

### System Update

:::warning
请不要使用 `apt-get upgrade` 命令更新系统，这会导致系统更新不完全，从而导致功能异常或缺失，甚至系统无法启动。  
应使用 `rsetup` 更新系统。
:::

本选项会更新你的系统中安装的软件包到最新状态。

由于 RadxaOS 提供了一些和上游 Debian 有所冲突的软件包，为了保证系统一致性，建议通过此方式进行系统更新。

### Change default boot target

设置系统默认启动到命令行界面或图形界面：

- **multi-user.target**: 命令行界面（CLI）
- **graphical.target**: 图形界面（桌面环境）

### Update Bootloader

本选项会更新根文件系统所在磁盘上的启动器。

### Update SPI Bootloader

本选项会更新 SPI Flash 上的启动器。

### Update eMMC Boot Bootloader

本选项会更新 eMMC Boot 分区上的启动器。

### Sleep / Hibernate Control

- **Enable Sleep / Hibernate**: 启用系统休眠功能
- **Disable Sleep / Hibernate**: 禁用系统休眠功能

更改后建议重启系统以使设置生效。

### Thermal governor

该模块用于调整风扇和其他温度选项。
一般来说，当使用直流风扇或不使用风扇时，选择 `power_allocator`，当使用 PWM 风扇时，选择 `step_wise`。

**注意：如果你将风扇直接连接到 GPIO 的 5V 和 GND 引脚上，它将不受控制且一直工作。**

### Configure DSI display mirroring

## Connectivity

该选项与网络设置类似。

选择 Network 选项后，会启动 `nmtui`（NetworkManager Text User Interface），提供完整的网络管理功能：

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         NetworkManager TUI                                  │
│                                                                             │
│  Please select an option                                                    │
│                                                                             │
│  Edit a connection                                                          │
│  Activate a connection                                                      │
│  Set system hostname                                                        │
│                                                                             │
│  Quit                                                                       │
│                                                                             │
│                                  <OK>                                       │
└─────────────────────────────────────────────────────────────────────────────┘
```

**网络管理功能**：

#### Edit a connection（编辑连接）

- 添加新的网络连接（Wi-Fi、以太网、VPN 等）
- 编辑现有连接的配置
- 删除不需要的连接
- 配置 IP 地址（静态/DHCP）
- 设置 DNS 服务器

#### Activate a connection（激活连接）

- 查看可用的网络连接
- 连接到 Wi-Fi 网络
- 激活或停用网络接口
- 查看连接状态

#### Set system hostname（设置主机名）

- 修改系统主机名

### Wi-Fi 配置步骤

1. 选择 `Activate a connection`
2. 选择你的 Wi-Fi 网络（会显示可用的 SSID 列表）
3. 输入 Wi-Fi 密码
4. 等待连接成功

### 有线网络配置

1. 选择 `Edit a connection`
2. 选择以太网接口
3. 配置 IP 地址模式：
   - **Automatic (DHCP)**: 自动获取 IP 地址
   - **Manual**: 手动配置静态 IP

:::tip

- 使用 `Tab` 键在不同选项间切换
- 使用 `Enter` 键确认选择
- 使用 `Esc` 键返回上一级菜单
- NetworkManager 配置通常会立即生效，无需重启
  :::

### Wi-Fi Country

设置 Wi-Fi 国家代码以符合当地法规。

**为什么需要设置 Wi-Fi 国家代码？**

- 不同国家/地区的 Wi-Fi 频道和功率限制不同
- 设置正确的国家代码可以：
  - 确保使用合法的 Wi-Fi 频道
  - 避免违反当地无线电法规
  - 获得更好的 Wi-Fi 性能与稳定性

:::warning
设置错误的国家代码可能导致：

- 无法连接到某些 Wi-Fi 网络
- Wi-Fi 信号变弱
- 违反当地法律法规
  :::

## 故障排除

### rsetup 启动失败

**问题**: 运行 `rsetup` 后无法启动或提示权限错误

**解决方案**:

1. 确保使用 `sudo rsetup` 或系统会自动提权
2. 检查终端是否支持当前的 TERM 类型，如果不支持尝试使用 `export TERM=xterm-256color` 设置环境变量
3. 尝试在不同的终端模拟器中运行

### 更新失败

**问题**: 系统更新过程中出现错误

**解决方案**:

1. 检查网络连接是否正常
2. 尝试更换 APT 源镜像
3. 检查系统时间是否正确（可能导致 HTTPS 证书验证失败）
4. 查看错误信息，必要时在社区寻求帮助

### Overlay 配置后无法启动

**问题**: 启用某个 overlay 后系统无法启动

**解决方案**:

1. 通过串口或 SSH 连接到系统（如果可能）
2. 编辑 `/boot/config.txt` 或 `/boot/uEnv.txt`，移除或注释掉问题的 overlay
3. 重启系统
4. 如果无法通过软件方式修复，可能需要从 SD 卡启动并修改配置文件

## 最佳实践

1. **定期更新**: 使用 `System Update` 定期更新系统，保持软件包为最新状态
2. **谨慎配置启动器**: 除非必要，不要随意更新或擦除启动器
3. **测试 Overlay**: 启用新的 overlay 前，先了解其功能和可能的影响
4. **备份重要数据**: 在进行重大配置更改前，备份重要数据
5. **记录更改**: 记录你所做的配置更改，便于故障排除
