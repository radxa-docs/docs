Yocto 项目 (YP) 是一个开源协作项目，可帮助开发人员创建基于 Linux 的定制系统。
Yocto 也可用于构建基于瑞芯微 SoC 设计的平台的系统。
下面将介绍基于瑞芯微 SoC 设计的瑞莎设备的 Yocto 构建。目前主要有两个版本：社区版本和厂商版本。

## 快速入门指南

### 步骤 1：选择 Yocto 版本
- **社区版本**：基于上游 meta-rockchip，支持更多设备，更新频率高
- **厂商版本**：基于瑞芯微官方 Yocto，提供更好的稳定性和硬件支持

### 步骤 2：准备构建环境
请参考 [构建系统文档](../build-system) 了解详细的硬件要求和环境搭建步骤。

### 步骤 3：获取源码
根据您选择的版本，执行相应的命令：

```bash
# 社区版本
mkdir ~/yocto-community && cd ~/yocto-community
repo init -u https://github.com/radxa/meta-rockchip.git -b scarthgap
repo sync

# 厂商版本
mkdir ~/yocto-manufacturer && cd ~/yocto-manufacturer
repo init -u https://github.com/radxa/yocto-manifests.git -b scarthgap
repo sync
```

### 步骤 4：配置和构建
1. 链接设备配置文件
2. 初始化构建环境
3. 开始构建

具体步骤请参考相关设备的构建文档。

### 步骤 5：烧录系统
构建成功后，您可以使用以下工具烧录系统：
- **Windows**: SDDiskTool
- **Linux/Mac**: dd 命令或 balenaEtcher

详细烧录步骤请参考 [安装系统到 MicroSD 卡](../system-to-sd) 或 [安装系统到 eMMC](../system-to-emmc)。

## 常见问题

### Q: 我应该选择社区版本还是厂商版本？
A: 如果您需要最新功能和更多设备支持，选择社区版本。如果您需要更好的稳定性和硬件支持，选择厂商版本。

### Q: 构建过程需要多长时间？
A: 首次构建通常需要 2-4 小时，取决于您的硬件配置和网络速度。后续构建会快很多，因为大部分包已经缓存。

### Q: 我可以自定义系统吗？
A: 是的，您可以：
1. 添加或删除软件包
2. 修改内核配置
3. 定制文件系统内容
4. 调整启动参数

### Q: 如何解决构建错误？
A: 常见的解决方法包括：
1. 确保所有依赖包已安装
2. 清理构建缓存：`bitbake -c cleansstate <recipe-name>`
3. 更新源码：`repo sync`
4. 检查网络连接
