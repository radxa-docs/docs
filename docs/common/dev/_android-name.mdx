## 准备工作：

- 一根 FAT(32) 格式的 SSD.
- 一条连接 PC 和 Radxa SOC 的 USB 数据线.
- Radxa SOC.
- Radxa SOC 对应的 Android spi nvme 镜像.
- Windows 平台需要 RkDevTool v2.93 或以上版本, Android12 rkr14 镜像需要 RkDevTool v3.13 或以上版本.
- Linux 平台需要 upgrade_tool v2.17 或以上版本.

## 烧录步骤

### Windows 烧录

1. 安装 SSD 到 Radxa SOC 对应的接口.

2. 打开 RkDevTool 烧录工具并点击升级固件，进入升级固件页面.

3. 按住板子 maskrom 按键，用数据线连接 PC 和 Radxa SOC，并观察 RkDevTool 工具底部是否出现 “MASKROM” 字样.

4. 点击 RkDevTool “固件”按键 找到当前 Radxa SOC 的 Android spi nvme 镜像.

5. 点击 RkDevTool “升级”按键，开始烧录固件并等待烧录完成.

### Linux 烧录

1. 安装 SSD 到 Radxa SOC 对应的接口.

2. 用数据线连接 PC 和 Radxa SOC.

3. 终端使用 ./upgrade_tool ld 并观察结果是否有 “maskrom” 字样.

4. 终端使用 ./upgrade_tool uf xxx.img (Radxa SOC 对应固件)，开始烧录固件并等待烧录完成.

## 常见问题

- 切换PCIE存储失败，loader不支持切换

  1. 检查 SSD 是否插入 Radxa SOC 对应的接口，有双接口的可以试试另一个接口.
  2. 检查 SSD 格式是否为 FAT32 格式.
