:::note
**注意：** 不同版本的镜像可能使用不同的默认用户名和密码。如果上述凭据无法登录，请尝试以下方法：

- 检查您下载的镜像的发布说明，查看是否有特定的默认凭据
- 尝试使用 `root` 用户（无需密码，直接按 Enter）
- 尝试通过串口控制台登录进行故障排除
- 对于某些基于 Debian 的镜像，可以尝试 `pi` / `raspberry`

如果仍然无法登录，可能是镜像下载或烧录过程中出现问题，请重新下载并烧录镜像。
:::
