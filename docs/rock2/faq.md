# ROCK 2 系列常见问题

## ROCK 2A 常见问题

### 问题：我的 ROCK 2A 在运行一段时间后无法启动

如果你的 ROCK 2A 在正常运行一段时间（例如一个月）后突然无法启动，即使更换了 SD 卡并重新安装系统仍然无效，可以尝试以下故障排除步骤：

#### 1. 电源检查
- **电源适配器**：确保使用的是官方的 12V/2A 电源适配器。不稳定的电源可能导致启动失败。
- **电源指示灯**：检查板上的红色电源指示灯是否亮起。如果不亮，可能是电源问题。
- **电源测试**：尝试更换一个已知良好的电源适配器，确保输出电压稳定。

#### 2. 硬件状态检查
- **过热问题**：检查板子是否过热。长期运行可能导致散热片脱落或风扇故障。
- **物理损坏**：检查板子是否有明显的物理损坏，如电容鼓包、芯片烧毁痕迹等。
- **连接松动**：检查所有连接线，包括电源线、显示屏线等，确保连接牢固。

#### 3. 启动诊断
- **串口调试**：通过 USB 转 TTL 串口线连接 UART2（40 针 GPIO 接口的引脚 8-TX、10-RX、6-GND），查看启动日志。
  - 波特率：1500000
  - 查看是否有明显的错误信息或卡在某个启动阶段。
- **LED 状态**：观察启动时的 LED 指示灯状态。正常启动时，蓝色 LED 会闪烁。

#### 4. 存储设备检查
- **eMMC 状态**：如果你使用的是 eMMC 版本，尝试重新烧录 eMMC 固件。
- **SD 卡质量**：即使更换了 SD 卡，也可能存在兼容性问题。尝试使用高质量的 Class 10 UHS-I SD 卡。

#### 5. 固件/软件恢复
- **固件更新**：尝试更新最新版本的引导程序（U-Boot）和内核。
- **最小系统启动**：移除所有外设（USB 设备、显示屏等），只保留电源和串口，尝试最小系统启动。

#### 6. 联系技术支持
如果以上步骤都无法解决问题：
- 在 GitHub 上提交详细的问题报告：https://github.com/radxa-docs/docs/issues
- 提供尽可能多的信息：
  - 购买渠道和购买时间
  - 故障发生前的使用情况
  - 已经尝试的解决方法
  - 串口日志截图或文本
- 联系官方客服，提供产品序列号和购买凭证

#### 常见可能原因
1. **电源管理芯片故障**：长期使用可能导致电源管理芯片损坏。
2. **eMMC 存储损坏**：如果是 eMMC 版本，存储芯片可能损坏。
3. **过热导致的元件老化**：散热不良可能导致关键元件性能下降。
4. **固件损坏**：系统更新或意外断电可能导致固件损坏。

> **注意**：这些问题通常需要专业技术支持。如果你不具备电子维修经验，建议联系官方售后服务。
