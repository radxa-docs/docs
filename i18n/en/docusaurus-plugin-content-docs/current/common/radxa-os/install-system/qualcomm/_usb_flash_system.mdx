## Download System Image

You need to visit the <a href={props?.download_page ?? '../../../download'}>Downloads</a> page to download the **latest system image**.

:::tip Image Note
The downloaded system image is a compressed file and needs to be extracted before use!
:::

## Flash System via USB

You need to copy the extracted system image file to the SPI boot firmware's {props.spi_path} directory, then open a terminal in that folder and run the following command to flash the system image:

<NewCodeBlock tip={`radxa@${props?.board ?? 'device'}$`} type="device">

```bash
edl-ng --loader <path-to-loader> --memory <storage-type> write-sector <start-sector> <path-to-image-file>
```

</NewCodeBlock>

Parameter Description:

- `<path-to-loader>`: Path to the bootloader, replace with the actual bootloader path.

- `<storage-type>`: Storage type, replace with the actual storage type.

- `<start-sector>`: Starting sector, replace with the actual starting sector.

- `<path-to-image-file>`: Path to the system image file, replace with the actual system image file path.

Example:

<code>{`edl-ng --loader ${props?.loader ?? "<path-to-loader>"} --memory ${
  props?.storage_type ?? "<storage-type>"
} write-sector ${props?.start_sector ?? "<start-sector>"} ${
  props?.image_file ?? "<path-to-image-file>"
}`}</code>

## How to tell whether flashing succeeded

After you run `edl-ng ... write-sector ...`, the terminal will first show write progress. If flashing succeeds, the progress will reach `100.0%`, then you should see output similar to:

```text
Writing: 100.0% (4096.00 / 4096.00 MiB) [22.31 MiB/s]
Successfully wrote 4096.00 MiB in 183.55s.
```

If you see output like this and the command returns to the shell prompt normally, the image write is generally complete.

## Common failure signs

If the terminal never shows `Successfully wrote ...`, or instead prints errors such as `Error:` / `Operation Error:` / `IO Error:` / `Platform Error:`, then the flash did not complete successfully. Common examples include:

```text
Error: Input file '/path/to/image.img' not found.
```

```text
Error: Sector range exceeds supported bounds.
```

```text
An unexpected error occurred in 'write-sector': <specific error message>
```

If flashing fails, first check:

1. Whether the board has actually entered EDL mode.
2. Whether the `prog_firehose_ddr.elf` path is correct.
3. Whether the storage type matches the target media (for example `ufs`, `Sdcc --slot 0`, or `nvme`).
4. Whether the image file has been fully extracted and the path is correct.
5. Whether the USB data cable and the board-to-host connection are stable.
